# ADR-009: Cognitive Divergence Alert Hard Rollback Protocol

## Status (状态)
Accepted (已接受)

## Context (背景)
**English:**
Signals ingested between June 23 and June 27, 2026, explicitly validate the necessity of absolute deterministic execution paths for agentic safety. Specifically, signal SIG_R1_003 (June 24) and subsequent analyses demonstrate that agent safety protocols must adopt hard rollback mechanisms when "cognitive divergence alerts" are triggered. This requires the system to mathematically bound topological entropy and formally reject non-deterministic hallucination chains before they commit to the system state.

**Chinese (中文):**
2026年6月23日至6月27日期间摄入的信号明确证实了智能体安全必须依赖绝对确定的执行路径。具体而言，信号 SIG_R1_003（6月24日）及后续分析表明，当触发“认知发散警报”时，智能体安全协议必须采用硬回滚机制。这要求系统在数学上界定拓扑熵的边界，并在非确定性幻觉链提交到系统状态之前将其正式拒绝。

## Decision (决策)
**English:**
We explicitly define the "Cognitive Divergence Alert" as a non-negotiable hard stop. If the convergence drill fails to reach a zero-entropy hash, or if topological entropy exceeds the defined boundary during the Gas phase (as identified in METH-004), the current reflection cycle is entirely invalidated. The system will perform an absolute SQLite SAVEPOINT rollback. The system is structurally prohibited from "re-reasoning" the faulty state; it must return to the last known deterministic execution baseline.

**Chinese (中文):**
我们明确将“认知发散警报”定义为不可协商的硬性停止。如果收敛演练未能达到零熵哈希，或者在气态（Gas）阶段拓扑熵超出了定义的边界（如 METH-004 中所定义），当前的反射周期将被完全失效。系统将执行绝对的 SQLite SAVEPOINT 回滚。系统在结构上被禁止“重新推理”错误状态；它必须返回到上一个已知的确定性执行基线。

## Architectural Constraints (架构约束)
1. **English:** The execution of a Hard Rollback must be deterministic, executing immediately without probabilistic review.
   **Chinese (中文):** 硬回滚的执行必须是确定性的，应立即执行且无需概率性审查。
2. **English:** The rejected state must be wiped from the continuum database and its delta logged strictly as 'NONE' in the dehydration cycle.
   **Chinese (中文):** 被拒绝的状态必须从连续体数据库中清除，并且其差分必须在脱水周期中严格记录为“NONE”。
