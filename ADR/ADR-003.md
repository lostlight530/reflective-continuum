# ADR-003: Cognitive Delta Protocol

## Status (状态)
Accepted (已接受)

## Context (背景)
To evaluate its own progress, the system needs to detect changes between different snapshots of its knowledge state. Measuring these differences using embeddings or neural similarities introduces subjective, non-deterministic drift.
为了评估自身的进展，系统需要检测其知识状态的不同快照之间的变化。使用嵌入或神经相似度来衡量这些差异会引入主观的、非确定性的漂移。

## Decision (决策)
We establish the Cognitive Delta Protocol, relying on three deterministic signals:
我们确立认知差分协议，依靠三个确定性信号：
1. **Structural Delta (结构差分):** SQL diff of the knowledge graph (nodes/edges added, removed, or modified).
   **结构差分：** 知识图谱的SQL差异（节点/边的增、删、改）。
2. **Rank Delta (排名差分):** Shifts in entity PageRank scores to identify significant structural mutations.
   **排名差分：** 实体PageRank分数的偏移，用于识别显著的结构突变。
3. **Semantic Delta (语义差分):** Drift in result sets from identical Full-Text Search (FTS5) queries across time slices.
   **语义差分：** 同一全文搜索（FTS5）查询在不同时间切片下结果集的漂移。

## Architectural Constraints (架构约束)
1. No vector databases or cosine similarity matching shall be used for state comparison.
   状态对比中不得使用任何向量数据库或余弦相似度匹配。
2. Delta thresholds must be mathematically defined prior to execution.
   差分阈值必须在执行前进行数学定义。

## Consequences (后果)
- **Positive:** Changes in system knowledge are highly explicit, traceable, and entirely reproducible.
  **正面：** 系统知识的变更是高度明确、可追踪且完全可重现的。
- **Negative:** Semantic nuances that are structurally identically expressed might be missed by purely relational and lexical methods.
  **负面：** 纯粹的关系和词法方法可能会遗漏结构表达相同但语义存在细微差别的变化。
